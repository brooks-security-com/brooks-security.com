# Operations Consulting

I'm a Site Reliability Engineer who treats infrastructure as software. If you're running things by hand, paying more for cloud than you should, or afraid to touch production because nobody's quite sure what will break — that's the kind of problem I like.

This very site is a small proof of the philosophy: the content, the AWS infrastructure, and the CI/CD that ships both live in one repository and cost about a dollar a month to run, with no servers to babysit. I bring that same cost-conscious, automate-everything approach to real production systems.

## Where I can help

- **Infrastructure as code** — Terraform-managed, PR-locked infrastructure so every change is reviewed, planned, and reversible. No more click-ops drift.
- **Configuration as code** — Ansible-driven fleet management; I've kept hundreds of EC2 assets consistent and patched this way.
- **CI/CD pipelines** — GitHub Actions workflows that validate, plan, and deploy with the right gates in the right places.
- **Cloud cost optimization** — keeping AWS best-practice architecture without the AWS-sized bill. I'll find what you're overpaying for and fix it without breaking what works.
- **Reliability & databases** — production PostgreSQL and Amazon Aurora administration, container workloads on ECS, and load balancing with ALB, Nginx, or Caddy depending on what the job actually needs.
- **Killing toil** — the manual, repeated work that eats your team's week, turned into Python and Bash automation that just runs.

## How I work

I automate the boring parts and document the rest. Manual processes don't scale and they don't survive turnover; encoded ones do. I also won't reach for the heaviest tool by reflex — sometimes the right answer is Caddy instead of an ALB, or a one-shot container instead of a standing service.

Independent consulting work runs through my practice, **Brooks Security LLC**.
