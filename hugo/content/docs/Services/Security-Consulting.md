# Security Consulting

I'm a CISSP who builds security programs that hold up to audits and to attackers — not binders that gather dust. Most engagements start the same way: an organization knows it needs to be more secure (often because a customer, regulator, or investor is asking), but doesn't have the in-house depth to stand up a real program. That's the gap I fill.

I've led a company through SOC 2 Type 1 and Type 2 — two years running with no adverse findings — by writing the policies, building the procedures, and wiring up the controls that make the framework true rather than performative. I've stood up a security program from nothing inside an early-stage startup, and I've delivered security and infrastructure work across medical, municipal, SaaS, critical infrastructure, government, education, and banking environments.

## Where I can help

- **Compliance & audit readiness** — SOC 2 and adjacent frameworks: policy and procedure authorship, control implementation, evidence automation, and getting you genuinely ready for the auditor instead of scrambling the week before.
- **Cloud security** — AWS hardening done as code. IAM administration and audit, least-privilege rebuilds, and controls enforced through Terraform and Ansible so they *stay* enforced.
- **Vulnerability management** — standing up real vuln-management workflows for Linux fleets, including regulated environments with tens of thousands of endpoints under management.
- **Security program development** — the foundational policies, procedures, and security-aware culture that turn "we should be more secure" into something operational.

## How I work

Automation first. A control a human has to remember to run is a control that eventually doesn't get run — so I'd rather encode it once, prove it in a pipeline, and move on. I also keep things honest: I'll tell you what actually moves your risk needle versus what's compliance theater.
