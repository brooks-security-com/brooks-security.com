# Annual Policy Review Checklist

The Information Security Policy requires that all ISP policies be reviewed at least annually. This checklist ensures the review is thorough and produces auditable evidence.

## How to use this checklist

1. Schedule the review. Allocate 1-2 hours per policy depending on complexity. Spread reviews across a quarter rather than doing everything in one week.
2. Assign a reviewer. The policy owner is responsible. For most policies, this is the Security Officer. For role-specific policies, the relevant department head may be the reviewer.
3. For each policy, work through the checklist. Document changes in the revision history table at the bottom of each policy.
4. After all policies are reviewed, present a summary to the oversight body (per the Charter for Oversight Responsibility).
5. Archive the completed checklists as evidence for auditors.

## Review Checklist

### For every policy

| Check | Yes | No | Notes |
|-------|-----|----|-------|
| Does the policy still reflect current organizational practice? | | | |
| Have any laws, regulations, or contractual requirements changed that affect this policy? | | | |
| Have any organizational changes (new departments, new products, new geographies) created gaps in this policy? | | | |
| Are all referenced companion policies and procedures still active and correctly named? | | | |
| Are all role references still accurate? (Has the CISO changed? Has a department been reorganized?) | | | |
| Are the Related Documents still accurate and complete? | | | |
| Is the policy still readable by its intended audience? (If a policy has grown too long, split content into the companion procedure.) | | | |
| Has the companion procedure been reviewed to ensure it reflects current implementation? | | | |
| Have all approved exceptions to this policy been reviewed and either closed or renewed? | | | |

### For policies with regulatory impact

| Check | Yes | No | Notes |

## Evidence to retain

After completing the annual review:

1. Archive a copy of each policy as it existed before the review (the "prior version").
2. Retain completed checklists for each policy reviewed.
3. If an oversight body meeting reviewed the results, retain the meeting minutes.
4. Update each policy's Revision History table with the review date and a summary of changes.
5. If no changes were made, still update the Revision History table: "Annual review conducted on [date]. No changes required."

This evidence package answers the auditor question: "How do you ensure your policies remain current?"
